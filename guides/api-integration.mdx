## Overview

The Advice Generator uses the [Advice Slip API](https://api.adviceslip.com/) to fetch random pieces of advice. This guide explains how the API integration works and how to handle API responses.

## API Endpoint

The application uses a single endpoint to fetch random advice:

```
https://api.adviceslip.com/advice
```

## Implementation

The API integration is implemented in the `Card` component at src/components/card.jsx:10-17:

<CodeGroup>

```jsx card.jsx
const handleAdvice = async () => {
    setIsLoading(true)
    await fetch("https://api.adviceslip.com/advice")
        .then(response => response.json())
        .then(data => {
            const { slip } = data
            setAdvice(slip)
            setIsLoading(false);
        })
}
```

</CodeGroup>

### State Management

The component uses React hooks to manage the advice data and loading state:

```jsx
const [advice, setAdvice] = useState({ id: 0, advice: "" })
const [isLoading, setIsLoading] = useState(true)
```

### Initial Load

The application fetches advice when the component mounts:

```jsx
useEffect(() => {
    handleAdvice()
}, [])
```

## API Response Format

The Advice Slip API returns JSON data in the following format:

<CodeGroup>

```json Example Response
{
  "slip": {
    "id": 117,
    "advice": "It is easy to sit up and take notice, what's difficult is getting up and taking action."
  }
}
```

```json Another Example
{
  "slip": {
    "id": 52,
    "advice": "Don't be afraid to ask questions."
  }
}
```

</CodeGroup>

### Response Properties

<ParamField path="slip" type="object">
  The main container object for the advice data
</ParamField>

<ParamField path="slip.id" type="number">
  Unique identifier for the advice
</ParamField>

<ParamField path="slip.advice" type="string">
  The advice text content
</ParamField>

## Loading States

The application tracks loading states to provide visual feedback:

<Steps>
  <Step title="Set Loading State">
    When fetching new advice, `setIsLoading(true)` is called to show the loading animation
  </Step>
  
  <Step title="Fetch Data">
    The API request is made using the native `fetch` API
  </Step>
  
  <Step title="Update State">
    Once data is received, the advice state is updated and loading is set to false
  </Step>
</Steps>

The loading state controls a CSS animation class on the button:

```jsx
<button 
  type="button" 
  onClick={() => handleAdvice()} 
  className={`${isLoading ? "load" : ""}`}
>
```

## Error Handling

<Warning>
The current implementation does not include explicit error handling. Consider adding try-catch blocks for production use:
</Warning>

```jsx
const handleAdvice = async () => {
    setIsLoading(true)
    try {
        const response = await fetch("https://api.adviceslip.com/advice")
        if (!response.ok) {
            throw new Error('Failed to fetch advice')
        }
        const data = await response.json()
        const { slip } = data
        setAdvice(slip)
    } catch (error) {
        console.error('Error fetching advice:', error)
        // Handle error state
    } finally {
        setIsLoading(false)
    }
}
```

## API Limitations

<Note>
The Advice Slip API may cache responses for a short period. If you're testing and getting the same advice repeatedly, this is normal behavior.
</Note>

## Testing the API

You can test the API directly using curl:

```bash
curl https://api.adviceslip.com/advice
```

Or in your browser's console:

```javascript
fetch('https://api.adviceslip.com/advice')
  .then(res => res.json())
  .then(data => console.log(data))
```
