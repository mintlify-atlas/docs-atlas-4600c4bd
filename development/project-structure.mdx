## Overview

The Advice Generator follows a standard Vite + React project structure with organized directories for components, styles, and assets.

## Root Directory

```
advice-generator/
├── public/              # Static assets
├── src/                 # Source code
├── design/              # Design files
├── index.html           # Entry HTML file
├── package.json         # Dependencies and scripts
├── vite.config.js       # Vite configuration
├── style-guide.md       # Design system guide
└── README.md            # Project documentation
```

## Source Directory (`src/`)

The `src/` directory contains all application source code:

```
src/
├── assets/              # Images and SVG files

│   ├── icon-dice.svg
│   ├── pattern-divider-desktop.svg
│   └── pattern-divider-mobile.svg

│   └── card.jsx
```

## Key Files

### main.jsx

The application entry point that sets up React and renders the root component:

```jsx src/main.jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import Card from './components/card'
import "./stylesheets/styles.scss"

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <Card />
  </React.StrictMode>
)
```

<Info>
  **Key responsibilities:**
  - Import and initialize React
  - Import the main Card component
  - Import global styles
  - Render the application to the DOM
  - Wrap in React.StrictMode for development checks
</Info>

### card.jsx

The main component that displays advice and handles API interactions:

```jsx src/components/card.jsx
import icon_dice from "../assets/icon-dice.svg"
import divisor from "../assets/pattern-divider-desktop.svg"
import { useEffect, useState } from "react"

const Card = () => {
  const [advice, setAdvice] = useState({ id: 0, advice: "" })
  const [isLoading, setIsLoading] = useState(true)

  const handleAdvice = async () => {
    setIsLoading(true)
    await fetch("https://api.adviceslip.com/advice")
      .then(response => response.json())
      .then(data => {
        const { slip } = data
        setAdvice(slip)
        setIsLoading(false)
      })
  }

  useEffect(() => {
    handleAdvice()
  }, [])

  return (
    // JSX markup...
  )
}

export default Card
```

<Card title="Component Features" icon="react">
  - **State Management**: Uses `useState` for advice data and loading states
  - **Side Effects**: Uses `useEffect` to fetch initial advice on mount
  - **API Integration**: Fetches advice from Advice Slip API
  - **Loading Animation**: Visual feedback during API calls
  - **Event Handling**: Click handler for generating new advice
</Card>

## Public Directory

The `public/` directory contains static assets served directly:

```
public/

```

<Note>
  Files in the `public/` directory are served at the root path and don't go through the build process.
</Note>

## Entry HTML File

The `index.html` file at the root serves as the entry point:

```html index.html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />

    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  </head>
  <body>
    <main id="root"></main>

```

## Build Output

When you run `npm run build`, Vite creates a production-ready build:

```
dist/
├── assets/              # Compiled and optimized assets
│   ├── index-[hash].js  # Bundled JavaScript
│   ├── index-[hash].css # Compiled CSS from SASS
│   └── [images]         # Optimized images
├── favicon.png          # Static assets from public/
└── index.html           # Optimized HTML with asset links
```

<Check>
  The build process:
  - Compiles SASS to CSS
  - Bundles and minifies JavaScript
  - Optimizes images
  - Generates content hashes for cache busting
  - Tree-shakes unused code
</Check>
